# Simple Bash Utils

**Описание:**
Проект посвящен разработке базовых утилит Bash `cat` и `grep` на языке C (C11). Цель — научиться структурному программированию, работе с текстовыми файлами и командной строкой в Linux.

---

## Содержание проекта

* **Утилита `cat`**

  * Отображение содержимого файлов
  * Объединение файлов
  * Создание новых файлов
  * Поддержка всех флагов GNU:

    * `-b`, `-e`, `-n`, `-s`, `-t`
  * Расположение исходников: `src/cat/`
  * Исполняемый файл: `s21_cat`

* **Утилита `grep`**

  * Поиск по шаблону в файлах
  * Поддержка основных флагов:

    * `-e`, `-i`, `-v`, `-c`, `-l`, `-n`
  * Дополнительно (для бонуса):

    * Флаги: `-h`, `-s`, `-f`, `-o`
    * Поддержка комбинаций флагов: `-iv`, `-in` и др.
  * Используемые библиотеки для регулярных выражений: `pcre` или `regex`
  * Расположение исходников: `src/grep/`
  * Исполняемый файл: `s21_grep`

* **Общие требования:**

  * Код в структурном стиле, соблюдение Google Style
  * Makefile с целями: `s21_cat`, `s21_grep`
  * Возможность переиспользования общих модулей через `src/common/`
  * Интеграционные тесты для всех флагов и вариантов ввода
  * Обработка ошибок не имеет строгой формулировки
  * Ввод через stdin необязателен

---

## История и справка

* `cat` появился в ранних версиях Unix (Version 1), заменил `pr` и PDP-7 утилиту.
* `grep` был создан Кеном Томпсоном для анализа текста «Записок Федералиста». Название происходит от команды `g/re/p` в редакторе `ed`.

---

## Особенности реализации

* Использовать только стандартные и POSIX.1-2017 библиотеки
* Избегать устаревших функций (legacy/obsolete)
* Структурный подход к разработке
* Возможность расширения через общие модули (`src/common/`)
* Сборка с помощью Makefile
* Исполняемые файлы должны работать с аргументами командной строки
