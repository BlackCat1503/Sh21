# s21_string+

**Описание:**
Разработка собственной реализации библиотеки `string.h` с дополнениями на языке C (C11). Цель — закрепить работу со строками, структурное программирование и расширить стандартные функции (sprintf, sscanf, специальные функции обработки строк).

---

## Основные задачи

1. **Библиотека string.h**

   * Реализация стандартных функций работы со строками:

     * Копирование: `strcpy`, `strncpy`, `memcpy`, `memmove`
     * Конкатенация: `strcat`, `strncat`
     * Сравнение: `strcmp`, `strncmp`, `memcmp`
     * Поиск: `strchr`, `strrchr`, `strstr`, `strpbrk`, `memchr`
     * Другие: `strlen`, `strspn`, `strcspn`, `strtok`, `strerror`
   * Функции должны работать с **z-строками** ASCII
   * Статическая библиотека: `s21_string.a`
   * Префикс функций: `s21_`

2. **Частичная реализация `sprintf`**

   * Поддержка: `c`, `d`, `i`, `f`, `s`, `u`, `%`
   * Флаги: `-`, `+`, пробел
   * Ширина: число
   * Точность: `.число`
   * Длина: `h`, `l`

3. **Дополнительно: модификаторы sprintf**

   * Поддержка: `g`, `G`, `e`, `E`, `x`, `X`, `o`, `n`, `p`
   * Флаги: `#`, `0`
   * Ширина и точность: `*`, `.*`
   * Длина: `L`

4. **Дополнительно: `sscanf`**

   * Полная поддержка формата, включая флаги, ширину, точность, длину и типы преобразования

5. **Дополнительно: специальные функции обработки строк**

   * `to_upper`, `to_lower`, `insert`, `trim`
   * Возвращают новые строки, обработанные по указанным правилам
   * Обрабатывают ошибки корректно (возврат `NULL`)

---

## Требования к разработке

* Язык: C11, компилятор gcc
* Стиль: Google Style
* Структурное программирование, исключение дублирования кода
* Запрещено использовать стандартную `string.h` (кроме тестов)
* Собственные списки ошибок для `strerror` (POSIX/macOS/Linux)
* Unit-тесты: библиотека Check, покрытие ≥80% функций
* Makefile с целями:

  * `all`, `clean`, `test`, `s21_string.a`, `gcov_report`
  * `gcov_report` формирует html-отчет с покрытием
